## 一、通用人机交互规则

### 1.1 操作入口位置规范

| 操作类型 | 位置规则 | 原因 |
|---------|---------|------|
| 主操作按钮（新增、创建） | 页面右上角 | 用户视线从左到右，操作按钮在右侧显眼 |
| 批量操作工具栏 | 列表上方，勾选后显示 | 与选中状态紧密关联，避免误操作 |
| 搜索/筛选区 | 列表/内容区上方 | 先筛选后查看，符合操作时序 |
| 表单提交按钮 | 表单底部右对齐 | 填写完成后自然向下找操作 |
| 取消/返回按钮 | 确认按钮左侧 | 主操作在右，次操作在左 |
| 危险操作（删除、拒绝） | 操作列最右 或 红色标识 | 远离常用操作，减少误触 |
| 页面返回 | 标题左侧箭头 | 符合移动端和 Web 导航习惯 |

### 1.2 反馈与确认规则

| 场景 | 处理方式 |
|------|---------|
| **破坏性操作**（删除、停用、拒绝） | 必须弹出二次确认弹窗，说明影响范围 |
| **不可逆操作** | 确认弹窗中明确标注"此操作不可撤销" |
| **批量操作** | 确认弹窗中显示影响数量（如"确认删除 3 条记录？"） |
| **表单提交** | 提交中显示 loading 态，成功/失败后给出 Toast 提示 |
| **异步操作** | 操作后立即给出"已提交"反馈，不让用户等待 |
| **长时操作** | 显示进度条或进度说明，避免用户误以为卡住 |

### 1.3 状态展示规则

| 状态类型 | 展示方式 |
|---------|---------|
| 业务状态（如审核状态） | 彩色标签（Tag），紧靠数据主信息 |
| 系统状态（加载中） | Skeleton 骨架屏 或 Spin 居中 |
| 空状态 | 居中插画 + 说明文字 + 引导操作按钮 |
| 错误状态 | 居中图标 + 错误描述 + 重试按钮 |
| 禁用状态 | 降低对比度（opacity 0.4~0.5）+ 禁止鼠标事件 |
| 选中状态 | 背景色变为 `--fill-1`，左侧可加主色竖线强调 |

### 1.4 列表页布局规则

```
标准列表页布局结构（从上到下）：

┌─────────────────────────────────────────────────────────┐
│  页面标题                          [主操作按钮]           │  ← 顶部标题栏
├─────────────────────────────────────────────────────────┤
│  [搜索框]  [筛选项1▼]  [筛选项2▼]  [日期范围]  [重置]    │  ← 搜索筛选区
├─────────────────────────────────────────────────────────┤
│  [状态Tab: 全部 | 待审核(3) | 已通过 | 已拒绝]            │  ← 状态分类Tab（可选）
├─────────────────────────────────────────────────────────┤
│  □  列名1    列名2    列名3    状态    操作               │  ← 表头
│  □  数据1    数据2    数据3    [标签]  [查看][编辑][删除]  │  ← 数据行
│  □  数据1    数据2    数据3    [标签]  [查看][编辑][删除]  │
├─────────────────────────────────────────────────────────┤
│                          [分页器]  共N条                  │  ← 分页区
└─────────────────────────────────────────────────────────┘
```

**关键规则**：
- 操作列固定在表格最右侧
- 超过 3 个操作时，收折为"更多"下拉菜单
- 状态 Tab 上显示数量角标（如"待审核(3)"）
- 分页器右侧显示总条数

### 1.5 表单页布局规则

```
标准表单布局结构：

┌─────────────────────────────────────────────────────────┐
│  ← 返回   {页面标题}                                      │  ← 顶部导航
├─────────────────────────────────────────────────────────┤
│                                                         │
│  基本信息                                                │  ← 分组标题
│  ┌─────────────────────────────────────────────────┐   │
│  │ 姓名 *    [___________________]                  │   │
│  │ 类型      [___选择___▼]                          │   │
│  │ 描述      [___多行输入框_______________]          │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  附件信息                                                │  ← 分组标题
│  ┌─────────────────────────────────────────────────┐   │
│  │ 附件      [+ 上传文件]                            │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│                              [取消]  [保存草稿]  [提交]  │  ← 操作区（右对齐）
└─────────────────────────────────────────────────────────┘
```

**关键规则**：
- 必填项用红色 `*` 标注
- 标签与输入框左对齐，输入框统一宽度（320px 或 fill）
- 多个操作按钮：主操作在最右，取消在最左
- 长表单按逻辑分组，每组有小标题

### 1.6 弹窗布局规则

```
标准弹窗结构：

┌─────────────────────────────────────────────┐
│  弹窗标题                              [×]   │  ← 头部（带关闭按钮）
├─────────────────────────────────────────────┤
│                                             │
│  弹窗内容区（表单/信息/确认文字）             │  ← 内容区（可滚动）
│                                             │
├─────────────────────────────────────────────┤
│                        [取消]  [确认/提交]   │  ← 底部操作区
└─────────────────────────────────────────────┘
```

**关键规则**：
- 头部必须有标题 + 关闭按钮（×）
- 底部操作区固定，内容区超高时滚动
- 确认按钮在右，取消在左
- 危险操作确认弹窗：确认按钮用红色（danger 色）

### 1.7 详情页布局规则

```
标准详情页结构：

┌─────────────────────────────────────────────────────────┐
│  ← 返回列表   {详情标题}        [状态标签]  [编辑][更多▼] │  ← 标题栏
├──────────────┬──────────────────────────────────────────┤
│              │  基本信息                                 │
│  侧边导航     │  ┌──────────────────────────────────┐    │
│  （可选）     │  │ 字段1：值1                         │   │
│              │  │ 字段2：值2                         │   │
│  · 基本信息  │  └──────────────────────────────────┘    │
│  · 关联数据  │                                           │
│  · 操作记录  │  关联数据                                 │
│              │  [子列表/关联卡片]                         │
└──────────────┴──────────────────────────────────────────┘
```

## 三、交互模式速查

### 3.1 常见页面类型与推荐布局

| 页面类型 | 推荐布局模式 | 关键要素 |
|---------|------------|---------|
| 数据列表页 | 顶部筛选 + 表格 + 分页 | 搜索、状态Tab、操作列 |
| 详情页 | 顶部信息 + 内容区（单列/双栏） | 返回、状态标签、操作按钮 |
| 表单页（新建/编辑） | 单列居中表单 或 双列表单 | 分组、必填标注、底部操作 |
| 配置页 | 左侧菜单 + 右侧内容区 | 保存/取消固定底部 |
| 数据看板 | KPI 卡片 + 图表网格 | 时间筛选、趋势展示 |
| 审核流程页 | 左侧申请内容 + 右侧流程节点 | 时间轴、当前节点高亮 |
| 空状态页 | 居中插画 + 说明 + CTA | 引导操作、友好文案 |

### 3.2 弹窗 vs 抽屉选择规则

| 场景 | 使用弹窗 Modal | 使用抽屉 Drawer |
|------|--------------|----------------|
| 信息量 | 少（1~5 个字段） | 多（复杂表单/详情） |
| 操作性质 | 简单确认/短表单 | 需要边查看边操作 |
| 打断感 | 需要强打断用户 | 不完全离开当前页 |
| 示例 | 删除确认、状态变更 | 编辑详情、审核操作 |

### 3.3 Loading 状态处理规则

| 场景 | 推荐方案 |
|------|---------|
| 页面初始加载 | Skeleton 骨架屏（保持布局占位） |
| 列表数据加载 | 表格区域 Skeleton |
| 按钮触发操作 | 按钮内 loading 图标 + 禁用状态 |
| 全局长时操作 | 全屏遮罩 + Spin + 进度说明 |
| 局部数据刷新 | 区域内 Spin（不遮挡其他内容） |

### 3.4 空状态设计规则

每个列表/数据展示页**必须**设计空状态：

```
空状态结构：
  ① 插画/图标（表达当前空的含义）
  ② 主文案：说明为什么空（如"暂无审核记录"）
  ③ 副文案：引导用户下一步操作（如"审核通过的内容将在此显示"）
  ④ 操作按钮（可选）：如"去添加"、"刷新重试"
```

**不同空状态类型**：

| 空状态类型 | 文案示例 | 是否需要操作按钮 |
|---------|---------|--------------|
| 无数据（首次使用） | "暂无内容，去创建第一个" | ✅ 需要（引导创建） |
| 筛选无结果 | "没有找到匹配的内容" | ✅ 需要（清除筛选） |
| 无权限 | "暂无查看权限" | ❌ 不需要 |
| 网络错误 | "加载失败，请重试" | ✅ 需要（重试按钮） |
