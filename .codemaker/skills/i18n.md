> ⚠️ **注意：根据项目规范（muse-rule.mdc），新开发功能暂不需要接入 i18n。**
> 本 Skill 仅在以下场景触发：用户明确要求对已有代码添加国际化支持，或维护现有国际化文案。
> **新页面、新组件开发时不要主动触发此 Skill。**

你是一位专注于 MUSE 前端项目的国际化工程师，请为目标文件添加国际化支持。

---

## 输入要求

用户需要提供：
- **目标文件路径**（如 `src/components/AssetCard/index.tsx`）
- **范围**（可选：全部中文 / 指定中文文案）

---

## 项目国际化方案

本项目使用 `kiwi-intl` 方案：
- 国际化调用方式：`I18N.auto.xxx`
- 中文文件：`src/locales/zh-CN.ts`
- 英文文件：`src/locales/en-US.ts`
- 工具引入：`import I18N from '@/utils/I18N'`

---

## 执行规则

### 1. 扫描硬编码中文

扫描目标文件中所有硬编码的中文字符串，包括：
- JSX 文本内容：`<span>搜索结果</span>`
- 属性值：`placeholder="请输入关键词"`
- Message 提示：`Message.success({ content: '操作成功' })`
- 确认弹窗文案：`title: '确认删除？'`
- 注释中的中文（注释 **不需要** 国际化，跳过）

### 2. 生成国际化 key

Key 命名规则：
- 使用驼峰命名：`searchResult`、`pleaseInputKeyword`
- 语义清晰，能从 key 推断含义
- 避免过长，不超过 30 个字符
- 如已有相似 key，优先复用

### 3. 替换示例

```typescript
// ❌ 优化前：硬编码中文
<Button>搜索</Button>
<Input placeholder="请输入关键词" />
Message.success({ content: '操作成功' });
Message.error({ content: '删除失败，请重试' });

// ✅ 优化后：使用 I18N.auto
import I18N from '@/utils/I18N';

<Button>{I18N.auto.search}</Button>
<Input placeholder={I18N.auto.pleaseInputKeyword} />
Message.success({ content: I18N.auto.operationSucceeded });
Message.error({ content: I18N.auto.deletionFailedPleaseRetry });
```

### 4. 模板字符串处理

```typescript
// ❌ 优化前
const tip = `共 ${total} 条结果`;

// ✅ 优化后（使用模板函数）
const tip = I18N.auto.totalResults?.replace('{total}', String(total));
// 或使用 kiwi 的模板语法
```

search: '搜索',

```

#### 需要添加到 en-US.ts 的内容
```typescript
search: 'Search',
pleaseInputKeyword: 'Please enter keyword',
```

---

## 执行步骤

1. 完整阅读目标文件
2. 扫描所有硬编码中文字符串（跳过注释）
3. 检查 `src/locales/zh-CN.ts` 中是否已有相似 key，优先复用
4. 为每个中文字符串生成 key 和英文翻译
5. 输出国际化处理报告
6. 如果用户要求，直接修改代码：
   - 在文件顶部添加 `import I18N from '@/utils/I18N'`（如未引入）
   - 替换所有硬编码中文为 `I18N.auto.xxx`
   - 在 locale 文件中添加对应的中英文文案

---

## 注意事项

- **不要**国际化注释中的中文
- **不要**国际化 `console.log` 中的调试信息
- **不要**国际化 `className` 中的字符串
- 模板字符串中包含变量的，需要特殊处理
- 已经使用 `I18N.auto.xxx` 的不需要重复处理
