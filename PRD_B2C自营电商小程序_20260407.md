## 1. 背景与目标

### 1.1 背景

随着微信生态流量红利的持续释放，越来越多的品牌选择通过微信小程序建立私域电商阵地。相比 H5 和 App，小程序具有即开即用、流量分享便捷、转化链路短等优势。

本平台定位为 B2C 自营商城，由平台方统一采购、销售实体商品，自主管控商品质量、价格与物流体验，目标是打造高信任度、高转化率的购物场景。

### 1.2 目标

**业务目标：**
- 上线后 3 个月内完成首批 SKU（库存单位）上架，支持完整下单链路
- 支持活动运营（优惠券、满减），提升客单价和复购率

**用户目标：**
- 用户可以在小程序内流畅完成"浏览商品 → 加购 → 下单 → 支付 → 查看物流"完整购物流程
- 让用户在整个购物过程中感到清晰、可信赖

**成功指标：**
- 下单转化率 ≥ 3%
- 支付完成率 ≥ 80%
- 订单投诉率 ≤ 0.5%

## 2. 用户故事

### 2.1 目标用户

| 用户类型  | 特征描述                                           |
|---------|-------------------------------------------------|
| 普通消费者 | 已关注品牌公众号/小程序，希望方便地购买平台自营商品         |
| 回头客    | 曾购买过，希望查看历史订单、再次购买、申请售后              |
| 活动用户  | 通过优惠券/分享链接进入，价格敏感，转化意愿强               |

### 2.2 用户场景

**场景一：日常购物**
> 作为一个普通消费者，
> 我希望浏览商品分类、查看商品详情、加入购物车后统一结算，
> 以便完成一次便捷的购物体验。

**场景二：活动参与**
> 作为活动用户，
> 我希望在下单时使用优惠券或享受满减优惠，
> 以便以更优惠的价格购买心仪商品。

**场景三：订单跟踪**
> 作为已付款用户，
> 我希望在"我的订单"中查看物流状态，
> 以便知道包裹什么时候到达。

### 2.3 用户旅程（核心购物链路）

```
进入小程序
   ↓
首页/分类浏览 → 搜索商品
   ↓
商品详情页（规格、价格、评价）
   ↓
加入购物车 / 立即购买
   ↓
填写/选择收货地址
   ↓
选择优惠券（如有）
   ↓
确认订单 → 微信支付
   ↓
支付成功页
   ↓
我的订单 → 查看物流
   ↓
确认收货 → （可选）发表评价
```

## 3. 功能需求

### 3.1 功能模块概览

| 模块       | 优先级 | 说明                            |
|----------|-------|-------------------------------|
| 首页       | P0    | Banner 轮播、商品推荐、活动入口         |
| 商品分类    | P0    | 一级/二级分类导航                    |
| 商品搜索    | P1    | 关键词搜索 + 筛选/排序                |
| 商品详情    | P0    | 图文详情、规格选择、库存展示             |
| 购物车     | P0    | 加购、改数量、删除、结算                |
| 订单结算    | P0    | 地址、优惠券、价格确认                  |
| 微信支付    | P0    | 调起微信支付，支持支付回调               |
| 我的订单    | P0    | 全部/待付款/待发货/待收货/已完成/售后      |
| 物流查询    | P1    | 集成快递100或快递鸟，展示物流轨迹          |
| 售后申请    | P1    | 退款/退货退款申请，填写原因与上传凭证       |
| 个人中心    | P0    | 收货地址、优惠券、积分（可选）             |
| 优惠券     | P1    | 满减券、折扣券、领券/使用             |
| 商家后台    | P0    | 商品管理、订单管理、库存管理（Web 端）     |

### 3.2 详细需求

#### 3.2.1 首页

- **Banner 轮播**：支持 3~5 张图片轮播，可配置跳转链接（商品详情/活动页）
- **商品推荐**：按销量或人工排序展示推荐商品，每行 2 个
- **活动专区**：可配置限时活动入口（如秒杀、满减活动），活动开始前显示倒计时
- **异常处理**：图片加载失败时展示占位图；网络异常时提示"加载失败，点击重试"

#### 3.2.2 商品分类

- 支持一级分类（左侧导航）+ 二级分类（右侧列表）
- 每个分类展示商品数量
- 点击商品卡片跳转到商品详情页

#### 3.2.3 商品详情页

- **商品图片**：支持多图横向滑动
- **商品信息**：商品名称、价格（划线价/现价）、销量、剩余库存（库存不足 20 件时展示"仅剩 XX 件"）
- **规格选择（SKU）**：支持多属性（如颜色、尺码），选择后实时更新价格和库存
- **商品详情**：富文本图文描述
- **用户评价**：展示最新评价，支持展开全部
- **加入购物车 / 立即购买**：按钮常驻底部
- **边界情况**：
  - 商品下架时展示"商品已下架"，按钮置灰
  - 库存为 0 时展示"暂时缺货"，按钮置灰

#### 3.2.4 购物车

- 展示所选商品（图片、名称、规格、数量、小计）
- 支持修改数量、删除单品、全选/反选
- 底部展示合计金额与"去结算"按钮
- 失效商品（下架/缺货）单独分组展示，支持一键清除

#### 3.2.5 订单结算页

- 收货地址选择/新增
- 商品清单（不可修改，仅展示）
- 优惠券选择（展示可用券列表）
- 运费展示（满 X 元免运费，或固定运费）
- 实付金额 = 商品总价 - 优惠 + 运费
- "提交订单"按钮调起微信支付

#### 3.2.6 微信支付

- 集成微信小程序官方支付 API（`wx.requestPayment`）
- 支付成功 → 跳转支付成功页（展示订单号、预计发货时间）
- 支付失败 → 展示失败提示，提供"重新支付"入口
- 支付超时（30 分钟未支付）→ 订单自动关闭

#### 3.2.7 我的订单

- 订单状态分 Tab：全部 / 待付款 / 待发货 / 待收货 / 已完成 / 退款/售后
- 每个订单卡片展示：商品缩略图、名称、规格、数量、实付金额、订单状态
- 操作按钮：
  - 待付款：立即支付 / 取消订单
  - 待发货：申请退款
  - 待收货：查看物流 / 确认收货
  - 已完成：再次购买 / 评价 / 申请售后

#### 3.2.8 售后申请

- 支持"仅退款"和"退货退款"两种类型
- 退货退款需填写退货原因（下拉选择）、补充说明（可选）、上传凭证图片（≤ 3 张）
- 提交后展示售后单号和处理进度

#### 3.2.9 个人中心

- 用户头像、昵称（来源微信授权）
- 收货地址管理（新增/编辑/删除/设为默认）
- 我的优惠券（未使用/已使用/已过期）
- 联系客服（跳转微信客服或显示客服二维码）

## 4. 非功能需求

### 4.1 性能要求

| 指标          | 目标值                     |
|-------------|--------------------------|
| 首屏加载时间     | ≤ 2 秒（4G 网络）            |
| 商品详情页加载   | ≤ 1.5 秒                  |
| 支付成功跳转延迟  | ≤ 1 秒                    |
| 高并发（活动期间）  | 支持 1000 TPS（待与研发确认）     |

### 4.2 兼容性

- 微信版本：≥ 8.0
- iOS：≥ 14，Android：≥ 9
- 小程序基础库版本：≥ 2.24.0

### 4.3 安全性

- 支付金额必须以服务端为准，客户端不得传递价格参数
- 用户地址、手机号等敏感信息加密存储
- 接口需鉴权（token 机制），防止未授权访问

### 4.4 数据埋点

| 事件名称               | 触发时机          | 携带参数                         | 用途            |
|--------------------|---------------|-----------------------------|---------------|
| page_view_home     | 首页展示          | page_name                   | PV/UV 统计      |
| goods_detail_view  | 商品详情页展示       | goods_id, goods_name        | 商品曝光分析        |
| add_to_cart        | 点击"加入购物车"     | goods_id, sku_id, quantity  | 加购行为分析        |
| order_submit       | 提交订单成功        | order_id, amount            | 下单转化率         |
| payment_success    | 支付成功          | order_id, amount, pay_type  | 支付转化率、GMV 统计  |
| coupon_use         | 使用优惠券         | coupon_id, coupon_type      | 优惠券效果分析       |
| after_sale_submit  | 提交售后申请        | order_id, reason_type       | 售后率分析         |

### 5.2 关键交互细节

| 场景             | 交互规范                                                    |

## 6. 验收标准

### 6.1 核心功能验收

- [ ] 用户可以通过分类/搜索找到商品，进入详情页查看完整信息
- [ ] 商品 SKU 选择后，价格与库存实时更新
- [ ] 加入购物车后，购物车角标数量正确更新
- [ ] 订单结算页正确展示商品金额、优惠券抵扣、运费和实付金额
- [ ] 调起微信支付成功后，订单状态变为"待发货"
- [ ] 支付超时 30 分钟后，订单自动关闭，库存回滚
- [ ] 用户可在"我的订单"中看到所有历史订单及当前状态
- [ ] 用户可发起售后申请，系统正确记录并变更订单状态

### 6.2 数据验收

- [ ] 关键埋点事件（加购、下单、支付）正确上报并可在数据平台查到
- [ ] GMV、订单数、支付转化率等核心指标数据准确

### 6.3 边界场景验收

- [ ] 商品库存为 0 时，详情页按钮置灰，无法加购
- [ ] 商品下架时，详情页提示"商品已下架"
- [ ] 网络断开时，页面展示友好的错误提示而非白屏
- [ ] 支付取消后，返回待支付状态，订单未关闭
- [ ] 优惠券使用后，不可重复使用同一张券

## 7. 风险与待确认事项

| 风险点                    | 等级 | 建议应对方案                              |
|------------------------|----|------------------------------------|
| 微信支付商户号申请周期较长         | 高  | 提前 2 周启动微信支付商户号申请                  |
| 实体商品需对接快递接口，物流商待定    | 中  | 尽早确定合作快递商，评估接口接入成本                |
| 高并发活动（如秒杀）可能导致超卖     | 高  | 库存扣减使用数据库事务+乐观锁，活动前压测              |
| 微信授权登录政策变动            | 低  | 关注微信官方公告，后端做登录态兜底方案               |
| 商品图片 CDN 成本            | 中  | 评估图片存储与 CDN 方案，建议使用对象存储（如腾讯 COS）   |
