## 页面状态机

### 1. 创建 Skill 弹窗（入口选择页）
*针对小白优化：将原本平铺的 4 个 Tab 改为“主次分明”的引导式界面，首推 AI 生成。*

- **正常态**：
  - **核心推荐区**：占据弹窗主要区域的大卡片【✨ 智能生成】，副标题“用自然语言描述需求，AI 帮你自动写代码配置”，卡片整体有轻微呼吸或高亮渐变效果，吸引视线。
  - **进阶选项区**：下方或侧边折叠显示“高级选项 / 手动创建”，包含三个小入口：
    - 【安装命令】（副标题：通过一行代码快速安装）
    - 【上传文件】（副标题：导入本地已打包的 Skill 配置）
    - 【手动填写】（副标题：一步步表单填写所有细节）
- **悬停态（Hover）**：鼠标移至“智能生成”大卡片，边框加粗/变色，卡片浮起，指针变手型。移至底部小入口，背景色加深。

### 2. 智能生成输入页（核心路径）
- **正常态**：顶部标题“智能生成 Skill”，带有返回图标（`< 返回选择`）。主体是一个大文本域，占位符“请描述这个 Skill 应该做什么、什么时候应用等，尽量描述全面来获取更好的结果”。右下角有字数统计（`0 / 10000`），底部是「开始生成」主按钮和取消按钮。
- **输入态**：文本域聚焦边框变色，字数实时更新。
- **验证失败态（Error）**：输入为空点击生成时，文本域边框变红，下方出现红色小字“请输入至少 X 个字符的需求描述”。
- **生成中（Loading）**：「开始生成」按钮变为“生成中...”，显示 Spinner，不可点击。弹窗主体不可操作，防止重复提交。
- **生成成功态**：Toast 提示“生成成功”，弹窗关闭，跳转到生成的配置预览或列表页（视具体业务而定，此处专注弹窗本身）。

### 3. 高级选项页（安装/上传/手动）
- **正常态**：顶部带返回图标。根据所选项展示对应的输入框或上传控件。
- **其他状态**：与现有系统类似，重点是保留“返回”机制，防止小白误点后不知所措。

## 用户操作路径
【小白推荐路径】点击「创建 Skill」 → 弹窗主推【智能生成】卡片 → 点击卡片进入输入页 → 填写需求 → 点击「开始生成」 → Toast 成功 → 弹窗关闭。
【误入纠正路径】点击「创建 Skill」 → 误点下方【手动填写】 → 发现看不懂表单 → 点击左上角【< 返回】 → 回到入口页重新选择【智能生成】。
【高级用户路径】点击「创建 Skill」 → 忽略大卡片，直接点击下方【安装命令】 → 粘贴命令 → 确定。

## 组件需求预估
- 弹窗容器（支持多步骤切换、顶部返回栏）
- 引导式大卡片（带渐变边框或图标）
- 折叠/次级入口链接组
- 文本域（带字数统计）
- 主次按钮组（主按钮需支持 Loading 态）
- Toast 提示组件
- 内联错误提示组件
