# Codemaker Dashboard — 产品需求文档（PRD）

> 文档版本：v1.0  
> 撰写日期：2026-04-01  
> 产品定位：网易 Codemaker 团队内部使用的本地桌面 AI 助手控制台

---

## 一、产品概述

### 1.1 产品背景

Codemaker Dashboard 是网易 Codemaker 团队为内部开发者打造的本地桌面 AI 助手图形化控制台，基于 macOS 平台运行。其核心职责是为 `codemaker serve`（运行在本地 `127.0.0.1:4000` 的 AI 推理服务）提供图形化前端，使开发者无需通过命令行即可与 AI 进行对话、管理 Agent、配置技能和 MCP 工具。

### 1.2 目标用户

- **主要用户**：网易内部使用 Codemaker 工具链的软件开发者
- **使用场景**：日常开发中与 AI 对话、管理自定义 Agent 和 Skill、配置 MCP 工具集成
- **用户特征**：具备一定技术背景，熟悉命令行和 AI 工具，有自定义 AI 助手能力的诉求

### 1.3 产品定位

| 维度 | 说明 |
|------|------|
| 运行环境 | macOS 本地桌面应用（Electron） |
| 服务依赖 | 需要 `codemaker serve` 在本地运行（`127.0.0.1:4000`） |
| 数据存储 | 会话/设置数据本地存储，不依赖云端 |
| 核心价值 | 图形化操控 AI 助手，降低使用门槛，提升配置效率 |

## 三、功能需求详述

### 3.1 登录页

#### 3.1.1 功能描述

用户首次使用时需要登录，登录成功后初始化本地数据并进入主界面。

#### 3.1.2 功能点

| 功能 | 描述 |
|------|------|
| 一键登录 | 点击"登录"按钮完成本地 OAuth 模拟登录，无需输入账密 |
| 种子数据初始化 | 登录成功后自动初始化默认 Agent（3个）和示例会话数据 |
| 登录状态保持 | 登录状态持久化，下次打开应用无需重新登录 |
| 登录态检测 | 应用启动时检测是否已登录，已登录则直接跳转聊天页 |

#### 3.1.3 需求边界

- 当前仅支持内部网易账号的本地模拟登录，不接入真实 OAuth 服务
- 登录失败时提示错误信息并允许重试

### 3.2 聊天页

聊天页是产品的核心页面，分为左侧会话列表侧边栏和右侧对话区域。

#### 3.2.1 会话管理（侧边栏）

| 功能 | 描述 |
|------|------|
| 会话列表 | 展示所有历史会话，按时间分组：置顶 / 今天 / 昨天 / 本周 / 更早 |
| 新建会话 | 点击"新建"按钮创建新会话，进入空白对话界面 |
| 切换会话 | 点击列表中任意会话可切换当前对话上下文 |
| 会话置顶 | 右键菜单可将会话置顶，置顶会话显示在列表顶部独立分组 |
| 会话重命名 | 双击会话标题可内联编辑重命名；右键菜单也提供重命名入口 |
| 会话删除 | 右键菜单删除会话，删除前需二次确认 |
| 会话导出 | 右键菜单支持导出为 Markdown 或 JSON 两种格式 |
| 会话搜索 | 快捷键 ⌘K 唤起搜索框，支持按关键词搜索历史会话标题 |
| 草稿暂存 | 切换会话时，当前输入框中未发送的内容自动暂存，切回时恢复 |

#### 3.2.2 对话输入区

| 功能 | 描述 |
|------|------|
| 文本输入 | 多行文本输入框，支持换行（Shift+Enter）和发送（Enter） |
| 图片附件 | 支持粘贴（Ctrl/Cmd+V）或拖拽上传图片，可多张 |
| URL 输入 | 输入框中可直接粘贴或输入 URL，系统自动识别后以文字卡片/标签形式展示（显示域名，如 `github.com`），替代原始 URL 文本；支持一条消息中包含多个 URL，每个独立显示为卡片；发送时优先由已配置的网页抓取类 MCP（如 Puppeteer）负责抓取，无可用 MCP 时前端直接抓取网页正文作为上下文拼入消息；不限制域名范围；前端抓取失败时卡片显示失败状态，降级为发送原始 URL，不阻断发送 |
| Skill 快捷调用 | 输入 `/` 触发 Skill 选择菜单，选择后将 Skill 触发词注入系统提示词 |
| Agent 选择 | 每次对话前可从下拉菜单切换当前使用的 Agent |
| 模型选择 | 可选择具体的 AI 模型，支持 20+ 个模型（含 Claude、GPT、Gemini、DeepSeek、Kimi、Qwen 等） |
| Token 用量提示 | 实时显示当前对话的上下文窗口使用百分比 |
| 发送限制 | AI 回复中（流式输出未完成时）禁止再次发送，输入框置灰 |

#### 3.2.3 消息展示区

| 功能 | 描述 |
|------|------|
| 流式输出 | AI 回复支持 SSE 实时流式渲染，逐字显示 |
| Markdown 渲染 | AI 回复完整支持 Markdown 格式（标题、列表、代码块、表格等） |
| 代码块高亮 | 代码块支持语法高亮，并提供一键复制按钮 |
| 工具调用过程展示 | AI 调用 MCP 工具时展示 ThinkingBlock，显示工具名称、调用参数和返回结果，可折叠 |
| 图片展示 | 用户发送的图片在消息气泡中内嵌展示 |
| AI 快捷选项 | AI 回复末尾自动识别列表项，生成快捷选项按钮，点击可快速发送对应内容 |
| 消息时间戳 | 每条消息显示发送时间 |

#### 3.2.4 消息操作

| 操作 | 适用范围 | 描述 |
|------|----------|------|
| 复制消息 | 所有消息 | 将消息内容复制到剪贴板 |
| 重新发送 | 用户消息 | 原样重新发送该条消息 |
| 编辑并重发 | 用户消息 | 进入编辑模式修改内容后重新发送，该消息之后的对话记录将被清除 |
| 重新生成 | AI 消息 | 基于同一上下文重新生成 AI 回复，替换当前回复 |
| 停止生成 | AI 消息（生成中） | 中断当前流式输出 |

#### 3.2.5 连接状态

| 状态 | 描述 |
|------|------|
| connecting | 正在连接 `codemaker serve` |
| connected | 已连接，可正常使用 |
| reconnecting | 连接断开后自动重连中 |
| disconnected | 连接失败，提示用户检查服务 |

顶部状态栏实时展示当前连接状态，连接异常时显示明显的错误提示。

#### 3.2.6 Provider 模式

聊天支持两种 AI 接入模式，可在设置页切换：

| 模式 | 说明 |
|------|------|
| Codemaker 模式（默认） | 对接本地 `codemaker serve`，使用 REST + SSE 协议，支持 Agent 调度和工具调用 |
| 直连模式 | 直接对接 OpenAI 兼容接口（通过本地代理解决 CORS），适用于自定义 API 场景 |

### 3.3 Agents 管理页

#### 3.3.1 功能描述

Agent 是具有特定角色和能力配置的 AI 实例，用户可以根据不同任务场景使用不同的 Agent。

#### 3.3.2 内置 Agent

系统内置 3 个默认 Agent，不可删除：

| Agent | 定位 |
|-------|------|
| Sisyphus | 通用编排 Agent，负责任务调度和协调 |
| Explorer | 代码探索专用，擅长阅读和理解代码库 |
| Builder | 全栈实现专用，擅长编写和修改代码 |

#### 3.3.3 功能点

| 功能 | 描述 |
|------|------|
| Agent 列表 | 展示所有 Agent，区分内置和自定义 |
| 创建 Agent | 填写名称、图标（emoji）、描述、关联 Skills、自动模式开关 |
| 编辑 Agent | 修改现有自定义 Agent 的所有属性 |
| 删除 Agent | 删除自定义 Agent，内置 Agent 不可删除 |
| Skills 关联 | 创建/编辑 Agent 时可选择关联哪些 Skills，Agent 执行时自动注入关联 Skill 的提示词 |
| 自动模式（autoMode） | 开启后 Agent 在执行时可自动调用工具，无需每步确认 |
| 磁盘文件同步 | 自动扫描 `opencode/agents/` 目录下的 `.md` 文件，将磁盘 Agent 同步到界面中展示 |
| 后端名称映射 | Agent 具有前端显示名（`name`）和后端 API 名称（`backendName`），两者可不同 |

#### 3.3.4 需求边界

- 自定义 Agent 上限为 **10 个**（不含内置的 3 个），达到上限后禁止继续创建，并提示用户
- 磁盘文件同步为单向同步（磁盘 → 界面），界面创建的 Agent 不写入磁盘
- 内置 Agent 可编辑（除名称外），但不可删除

### 3.4 Skills 管理页

#### 3.4.1 功能描述

Skill 是可注入系统提示词的技能包，每个 Skill 包含触发词、说明文档和标签。在对话时可通过 `/` 命令快速激活某个 Skill。

#### 3.4.2 安装方式

| 安装方式 | 描述 |
|----------|------|
| 安装命令 | 粘贴或输入 shell 安装命令，系统执行后自动加载新 Skill |
| 上传文件 | 上传 `.md` 格式的 Skill 定义文件 |
| 手动填写 | 在界面中逐项填写 Skill 属性（名称、描述、说明文档、标签、触发词） |

#### 3.4.3 功能点

| 功能 | 描述 |
|------|------|
| Skill 列表 | 展示所有已安装的 Skill，包含名称、描述、标签 |
| 启用/禁用 | 每个 Skill 可单独启用或禁用，禁用后不在 `/` 命令菜单中出现 |
| 编辑 Skill | 修改 Skill 的所有属性 |
| 删除 Skill | 删除 Skill，同时删除对应磁盘文件 |
| Skill Hub 入口 | 页面提供快捷入口跳转到 Skill Hub（`https://skills.netease.com`），供用户发现和安装社区 Skill |
| 磁盘持久化 | Skill 数据持久化存储在磁盘 `opencode/skills/` 目录的 `.md` 文件中 |

#### 3.4.4 需求边界

- Skill 触发词为注入系统提示词的关键字符串，一个 Skill 可有多个触发词
- Skill 的说明文档（README）支持 Markdown 格式
- Skill 文件来源与界面数据保持同步，手动修改磁盘文件后重启应用可加载变更

### 3.5 MCP 工具管理页

#### 3.5.1 功能描述

MCP（Model Context Protocol）是 AI 模型调用外部工具的标准协议。用户可以在此页面管理 AI 可调用的工具服务，包括本地进程型工具和远程 HTTP 型工具。

#### 3.5.2 内置 MCP（只读）

系统内置以下 MCP，用户不可删除或修改，仅可查看状态：

| MCP 名称 | 功能 |
|----------|------|
| Web Search（Exa.ai） | 互联网搜索能力 |
| Context7 | 技术文档查询 |
| Grep App | GitHub 代码搜索 |

#### 3.5.3 添加自定义 MCP

**智能安装**（推荐方式）：
- 用户粘贴任意格式的 MCP 配置信息（JSON 片段、URL、Token 字符串等）
- 系统自动识别格式并解析出必要配置项
- 支持识别的格式：标准 JSON 配置、HTTP URL、Bearer Token 等

**手动填写**：
- 本地 MCP：填写命令（command）、命令参数（args）、环境变量（env）
- 远程 MCP：填写服务 URL 和请求 Headers

**MCP 模板库**：
提供以下常用 MCP 的一键模板，选择后预填配置：

| 模板 | 用途 |
|------|------|
| Filesystem | 本地文件系统读写 |
| GitHub | GitHub 仓库操作 |
| PostgreSQL | 数据库查询 |
| Brave Search | 网络搜索 |
| Puppeteer | 浏览器自动化 |
| Slack | Slack 消息集成 |

#### 3.5.4 功能点

| 功能 | 描述 |
|------|------|
| MCP 列表 | 展示所有 MCP（内置 + 自定义），显示名称、类型（本地/远程）、状态 |
| 实时状态展示 | 轮询各 MCP 的连接状态，实时显示：connected / disconnected / connecting / needs_auth / failed |
| 启用/禁用 | 自定义 MCP 可单独启用或禁用 |
| 编辑 MCP | 修改自定义 MCP 的配置 |
| 删除 MCP | 删除自定义 MCP |
| 认证引导 | 状态为 `needs_auth` 时，引导用户完成认证操作 |
| 配置持久化 | MCP 配置保存到 `~/.config/codemaker-dashboard/opencode/opencode.json` |

#### 3.5.5 需求边界

- 内置 MCP 只可查看，不可编辑或删除
- 本地 MCP 依赖对应的命令行工具已安装在系统中
- 远程 MCP 需要网络可达
- 状态轮询周期为固定间隔（非实时推送）
- MCP 配置变更后需重启 `codemaker serve` 才能生效（系统提示）

## 四、全局交互规范

### 4.1 导航结构

- 左侧全局侧边栏包含：聊天、Agents、Skills、MCP、设置 五个一级导航入口
- 聊天导航下内嵌会话列表，其余导航点击后直接进入对应管理页
- 侧边栏支持折叠/展开

### 4.2 快捷键

| 快捷键 | 功能 |
|--------|------|
| ⌘K | 唤起会话搜索 |
| Enter | 发送消息 |
| Shift+Enter | 消息输入框内换行 |
| Ctrl/Cmd+V | 粘贴图片到输入框 |

### 4.3 右键菜单

聊天页会话列表项支持右键菜单，提供：
- 置顶 / 取消置顶
- 重命名
- 导出（Markdown / JSON）
- 删除

### 4.4 空状态

- 无会话时，聊天区域展示欢迎引导界面，提示用户新建会话
- 无 Agent 时，Agents 页展示引导创建提示
- 无 Skill 时，Skills 页展示安装引导和 Skill Hub 入口
- 无自定义 MCP 时，MCP 页展示内置 MCP 列表和添加引导

### 4.5 加载与错误处理

- API 请求中展示 loading 状态
- 请求失败时展示错误提示，支持重试
- `codemaker serve` 不可用时，聊天功能整体降级，展示服务不可用提示

## 六、非功能性需求

### 6.1 性能

- 流式输出（SSE）渲染延迟不超过 100ms
- 会话列表加载（含 100 条以上历史）响应时间不超过 500ms
- MCP 状态轮询不影响聊天交互性能

### 6.2 可靠性

- `codemaker serve` 意外退出时，Dashboard 自动尝试重启服务
- Config Proxy 服务（:5175）随 Dashboard 启动，Dashboard 退出时自动停止
- 网络断开时前端状态明确提示，不出现无响应的假死状态

### 6.3 兼容性

- 仅支持 macOS 平台
- 依赖本地安装 `codemaker` CLI 工具链

### 6.4 安全

- API Key（直连模式）本地明文存储于 localStorage，不上传服务器
- MCP 配置中的环境变量（含 Token 等敏感信息）存储于本地配置文件，不传输云端

## 八、已知边界与限制

1. **仅限 macOS**：当前版本不支持 Windows 和 Linux
2. **强依赖本地服务**：`codemaker serve` 不运行时，聊天功能不可用
3. **无云端同步**：所有数据仅本地存储，换设备需手动迁移
4. **模型依赖网络**：即使使用 Codemaker 模式，AI 推理仍需访问网络
5. **图片附件限制**：仅支持通过粘贴或上传方式添加图片，不支持从 URL 引用
6. **Skill 数量无硬性限制**：但过多 Skill 可能导致系统提示词超出模型 Token 上限
7. **会话历史无服务端备份**：清空数据操作不可恢复
