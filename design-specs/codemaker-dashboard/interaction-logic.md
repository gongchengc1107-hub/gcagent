## 全局布局结构

```
┌──────────────────────────────────────────────────────────────┐
│  标题栏（macOS 原生窗口控制 + 应用名）                         │
├────────┬─────────────────────────────────────────────────────┤
│        │                                                      │
│  全局  │              主内容区域                              │
│  左侧  │  （根据当前路由渲染对应页面）                         │
│  导航  │                                                      │
│  栏    │                                                      │
│        │                                                      │
├────────┴─────────────────────────────────────────────────────┤
│  底部状态栏（连接状态 + MCP 状态 + codemaker serve 状态）      │
└──────────────────────────────────────────────────────────────┘
```

### 全局左侧导航栏

- **宽度**：展开态 220px，折叠态 56px
- **折叠行为**：点击折叠按钮切换；折叠时仅显示图标
- **导航项**（从上到下）：
  1. 聊天（图标 + 文字）— 选中时展开会话列表子区域
  2. Agents
  3. Skills
  4. MCP
  5. （底部）设置
- **聊天项特殊行为**：选中聊天时，导航栏内嵌展示会话列表（见聊天页详述）

### 底部状态栏

- **高度**：28px
- **内容**（从左到右）：
  - codemaker serve 状态：`● 服务运行中` / `● 服务未运行` / `● 重连中`
  - MCP 汇总状态：`● N 个工具已连接` / `⚠ 1 个 MCP 连接失败`
  - 右侧留白
- **点击行为**：点击 MCP 状态 → 弹出 Popover 展示各 MCP 连接详情

## 页面状态机

---

### 1. 登录页

**正常态（未登录）**：
- 居中布局，展示产品 Logo + 应用名「Codemaker Dashboard」
- 副标题：「网易 Codemaker 本地 AI 助手控制台」
- 主按钮：「使用网易账号登录」
- 无任何输入框（一键登录）

**加载态（登录中）**：
- 按钮变为 loading 状态（spinner + 文字「登录中...」）
- 按钮不可重复点击
- 显示步骤提示：「正在初始化数据...」

**错误态（登录失败）**：
- 按钮恢复可点击
- 按钮下方展示红色错误提示文字（如「登录失败，请重试」）
- 不清除按钮，允许用户重试

**已登录态（自动跳过）**：
- 应用启动时检测到已登录 → 直接跳转聊天页，登录页不展示

### 2. 聊天页

聊天页为「左侧会话列表 + 右侧对话区」双栏布局。

#### 2A. 会话列表侧边栏（内嵌于全局导航栏展开区）

**宽度**：240px（固定，不可拖拽调整）

**正常态（有会话）**：
- 顶部：「新建会话」按钮（`+` 图标 + 文字）
- 顶部：快捷键提示「⌘K 搜索」
- 会话列表按时间分组（Label 为灰色小字）：
  - 置顶（有置顶会话时展示此分组）
  - 今天
  - 昨天
  - 本周
  - 更早
- 每条会话项：
  - 会话标题（单行截断）
  - Hover 时显示：「...」更多菜单按钮
  - 选中态：左侧 4px 高亮线 + 背景浅色

**空态（无任何会话）**：
- 仅显示「新建会话」按钮
- 列表区域显示：灰色提示文字「暂无会话记录，点击上方按钮开始」

**搜索态（⌘K 唤起）**：
- 顶部展示搜索输入框（自动聚焦）
- 列表实时过滤，按关键词匹配会话标题
- 无结果时显示：「未找到相关会话」
- 按 Esc 关闭搜索，恢复列表

**会话项右键菜单**：
```
├── 置顶（已置顶则显示「取消置顶」）
├── 重命名
├── 导出
│   ├── 导出为 Markdown
│   └── 导出为 JSON
└── 删除（触发二次确认）
```

**内联重命名态**：
- 双击会话标题 → 会话标题变为可编辑输入框（inline edit）
- Enter 确认 / Esc 取消
- 空内容不允许确认

**删除确认弹窗**：
- 标题：「删除会话」
- 描述：「此操作不可恢复，会话记录将被永久删除。」
- 操作：「取消」+ 「确认删除」（红色 / danger 按钮）

#### 2B. 对话区域（右侧主区域）

**空状态（新建会话 / 无历史消息）**：
- 居中展示欢迎引导界面：
  - 大号文字：「你好，我是 Codemaker」
  - 副文字：「选择一个 Agent 开始对话，或直接输入问题」
  - 快捷入口：展示 3 个常用 Skill 快捷按钮（如「代码审查」「需求分析」「写单测」）
- 底部正常显示输入区域

**正常对话态（有历史消息）**：
- 消息列表区（可滚动，新消息在底部）
- 每条消息：
  - 用户消息：右对齐气泡（含头像/首字母）
  - AI 消息：左对齐（含 Agent 名称/图标）
  - 消息时间戳（悬浮或始终展示）
- 消息操作（Hover 或点击消息时出现操作栏）：
  - 用户消息：复制 / 重新发送 / 编辑并重发
  - AI 消息：复制 / 重新生成

**AI 响应中态**：
- 输入区域整体置灰，禁止发送
- 消息列表底部展示「正在生成...」的 typing indicator
- 流式文字逐字输出
- AI 消息底部显示「停止生成」按钮

**工具调用展示态**（AI 响应中/后）：
- ThinkingBlock（推理过程）：
  - 默认折叠，显示「● 正在思考...」（执行中）或「✓ 已思考 Xs」（完成）
  - 点击展开查看完整推理文本
  - 背景色：轻紫色（`bg-purple-50`）
- ToolCall Block（工具调用）：
  - 每次调用单独一块
  - 展示：工具名称 + 关键参数（折叠显示）+ 调用结果（折叠显示）
  - 执行中：橙色 spinner + 「正在调用 {工具名}...」
  - 执行完成：绿色 ✓ + 工具名 + 「返回 N 条结果」
  - 点击可展开查看完整参数和结果
  - 背景色：轻橙色（`bg-orange-50`）
- AI 最终回复跟随在所有 ToolCall Block 之后展示

**AI 快捷选项**（AI 回复末尾）：
- AI 回复中如包含列表选项，自动识别并在回复末尾生成点击按钮
- 点击按钮 = 发送对应内容

**服务不可用态**（codemaker serve 断线）：
- 对话区顶部展示全宽横幅：「⚠ codemaker serve 未运行，聊天功能不可用」+ 「前往设置」按钮
- 输入区域置灰

**编辑并重发态**：
- 点击「编辑并重发」→ 用户消息变为可编辑输入框
- 该消息之后的所有对话记录显示为「将被删除」的半透明状态
- 操作：「取消」+ 「重新发送」
- 重新发送后，该条消息之后的记录清除，重新生成

#### 2C. 输入区域

**正常态**：

```
┌────────────────────────────────────────────────────────────┐
│  （图片预览区，有附件时展示）                                │
│  [img thumbnail ×] [img thumbnail ×]                       │
│  （URL 卡片区，有 URL 时展示）                               │
│  [github.com ×] [notion.so ×]                              │
├────────────────────────────────────────────────────────────┤
│  文本输入框（多行，自动扩展，min 48px，max 200px）           │
│  placeholder：「发送消息... (Enter 发送，Shift+Enter 换行)」 │
└────────────────────────────────────────────────────────────┘
  [📎 附件]  [/ Skill]  [Token: 23%]   [Agent▼]  [模型▼]  [⬆]
```

**底部工具栏**（从左到右）：
- 📎 附件按钮：点击选择本地图片文件
- `/` Skill 按钮：点击唤起 Skill 选择菜单（等同于输入 `/`）
- Token 用量：「上下文 23%」（实时更新）
- Agent 选择器（下拉）：显示当前 Agent 名称
- 模型选择器（下拉）：显示当前模型名称（截断显示）
- 发送按钮：蓝色，Enter 快捷键

**Skill 选择菜单（`/` 触发）**：
- 弹出方向：向上（Popover）
- 展示已启用的 Skill 列表
- 支持继续输入过滤
- 选中后将 Skill 触发词注入，菜单关闭

**Agent 下拉菜单**：
- 展示所有 Agent（内置 + 自定义）
- 内置 Agent 有特殊标识
- 当前选中项有 ✓ 标记

**模型下拉菜单**：
- 分组展示：Claude / GPT / Gemini / DeepSeek / Kimi / Qwen 等
- 支持搜索过滤
- 20+ 个模型

**URL 卡片**：
- 粘贴或输入 URL 后，自动识别，以标签卡片形式展示
- 卡片样式：域名 favicon + 域名文字（如 `github.com`）+ × 删除按钮
- 前端抓取失败时卡片显示「⚠ 抓取失败」状态，仍可发送（降级为原始 URL）

**图片附件预览**：
- 缩略图 + × 删除按钮
- 支持拖拽上传 / Ctrl+V 粘贴

**置灰态（AI 回复中）**：
- 整个输入区背景变暗
- 发送按钮不可点击
- 工具栏按钮不可点击
- 仅「停止生成」按钮可用（显示在输入框右下角）

#### 2D. 连接状态（顶部状态栏）

聊天页顶部有专属状态栏（高度 32px）：

| 状态 | 展示内容 | 颜色 |
|------|---------|------|
| connecting | `● 正在连接 codemaker serve...` | 蓝色 + 动画 |
| connected | `● 已连接` | 绿色（短暂显示后收起） |
| reconnecting | `● 重连中...` | 橙色 + 动画 |
| disconnected | `● 连接失败 — 请检查 codemaker serve 是否运行` + 「重试」按钮 | 红色 |

### 3. Agents 管理页

**正常态（有 Agent）**：
- 页面顶部：标题「Agents」+ 右侧「+ 创建 Agent」按钮
- 列表分两组（以分割线区分）：
  - **内置 Agent**（3 个）：Sisyphus / Explorer / Builder，带「内置」标签，无删除按钮
  - **自定义 Agent**：用户创建，有编辑/删除操作
- 每个 Agent 卡片/列表项展示：
  - Emoji 图标 + 名称 + 描述（单行截断）
  - 关联 Skills 数量标签（「3 个 Skills」）
  - 自动模式状态（Badge：「自动模式」）
  - 内置 Agent：仅「编辑」操作（名称不可编辑）
  - 自定义 Agent：「编辑」+「删除」操作

**空态（无自定义 Agent）**：
- 内置 Agent 组正常展示
- 自定义 Agent 组显示空状态提示：「还没有自定义 Agent」+ 「+ 创建 Agent」引导按钮

**达到上限态（自定义 Agent = 10 个）**：
- 「+ 创建 Agent」按钮置灰
- Hover 提示：「已达到 10 个自定义 Agent 上限」

**删除确认弹窗**：
- 标题：「删除 Agent」
- 描述：「删除后不可恢复，与该 Agent 相关的会话上下文不受影响。」
- 操作：「取消」+ 「确认删除」（danger）

**Agent 创建/编辑 Modal**：
- 标题：「创建 Agent」/ 「编辑 Agent」
- 字段：
  - 图标（Emoji 选择器）— 必填
  - 名称 — 必填（编辑内置 Agent 时此字段禁用）
  - 描述 — 选填
  - 关联 Skills（多选，展示 Skill 列表供勾选）
  - 自动模式开关（Toggle）
- 操作：「取消」+ 「保存」

### 4. Skills 管理页

**正常态（有 Skill）**：
- 页面顶部：标题「Skills」+ 右侧「+ 安装 Skill」按钮 + 「前往 Skill Hub」链接
- Skill 列表（卡片网格或列表形式）：
  - Skill 名称 + 描述（多行）
  - 标签（Tags，多个）
  - 触发词列表（`/trigger1` `/ trigger2`）
  - 启用/禁用 Toggle
  - 「编辑」+「删除」操作

**空态**：
- 展示大号引导区域：
  - 文案：「还没有安装任何 Skill」
  - 「+ 安装 Skill」按钮
  - 「前往 Skill Hub 发现更多 →」链接

**Skill 安装 Modal**：
- 标题：「安装 Skill」
- 三个 Tab：
  - **安装命令**：粘贴 shell 命令，「执行安装」按钮
    - 执行中：loading + 进度描述
    - 执行成功：「✓ 安装成功，Skill 已加载」
    - 执行失败：红色错误信息 + 允许重试
  - **上传文件**：文件拖拽区（`.md` 格式），点击或拖拽选择
  - **手动填写**：表单字段（名称、描述、说明文档、标签、触发词）

**Skill 编辑 Modal**：
- 同手动填写 Tab 的表单，回填已有数据

**禁用态（Skill 已禁用）**：
- 卡片样式半透明/变灰
- Toggle 显示为关闭状态
- 禁用的 Skill 不出现在输入框 `/` 菜单中

### 5. MCP 工具管理页

**正常态**：
- 页面顶部：标题「MCP 工具」+ 右侧「+ 添加 MCP」按钮
- 列表分两组：
  - **内置 MCP**（3 个）：Web Search / Context7 / Grep App，只读，无编辑/删除
  - **自定义 MCP**：可编辑/删除/启用禁用

**每个 MCP 卡片/列表项**：
- MCP 名称
- 类型标签：「本地」/ 「远程」
- 连接状态圆点 + 文字：
  - `● 已连接`（绿）
  - `● 连接中`（蓝 + 动画）
  - `● 连接失败`（红）
  - `● 需要认证`（橙）
  - `● 已禁用`（灰）
- 可用工具数量（已连接时显示）：「12 个工具」
- 自定义 MCP：启用/禁用 Toggle + 「配置」+ 「删除」按钮
- 内置 MCP：无操作按钮

**状态为 needs_auth 时**：
- 状态区域额外展示：「需要认证」橙色标签 + 「去认证 →」按钮
- 点击「去认证」→ 打开系统浏览器完成 OAuth 等认证流程

**空态（无自定义 MCP）**：
- 内置 MCP 组正常展示
- 自定义 MCP 组：引导卡片「+ 添加你的第一个 MCP 工具」

**配置变更提示**：
- 修改/删除/添加 MCP 后，页面顶部展示横幅：
  - 「MCP 配置已更新，需重启 codemaker serve 才能生效」
  - 操作按钮：「立即重启」

**MCP 添加 Modal**：
- 标题：「添加 MCP 工具」
- 三种安装方式（Tab 或步骤选择）：
  1. **智能安装**（推荐，默认激活）：
     - 大号文本区域：「粘贴 MCP 配置信息（支持 JSON、URL、Token 等格式）」
     - 「解析」按钮 → 自动识别后展示解析结果预览
     - 确认后保存
  2. **手动填写**：
     - 类型选择：本地 / 远程
     - 本地：名称 + 命令（command）+ 参数（args）+ 环境变量（env, key-value 列表）
     - 远程：名称 + URL + Headers（key-value 列表）
  3. **模板库**：
     - 6 个模板卡片（Filesystem / GitHub / PostgreSQL / Brave Search / Puppeteer / Slack）
     - 点击选中模板 → 自动填入预设配置 → 进入「手动填写」Tab 补充填写

### 6. 设置页

设置页为「左侧设置分类菜单 + 右侧内容区」双栏布局。

**分类菜单**（左侧，宽度 180px）：
- 账户
- AI Provider
- 消耗统计
- 外观
- 数据管理

#### 6.1 账户设置

**正常态**：
- 头像（圆形）+ 用户名 + 网易内部邮箱
- 「退出登录」按钮（位于账户信息卡片下方）

**退出登录确认弹窗**：
- 标题：「退出登录」
- 描述：「退出后本地登录状态将被清除，下次使用需重新登录。」
- 操作：「取消」+ 「确认退出」

#### 6.2 AI Provider 设置

**正常态（Codemaker 模式）**：
- Provider 模式切换 Radio/SegmentedControl：「Codemaker 模式（推荐）」/ 「直连模式」
- Codemaker 模式面板：
  - codemaker serve 运行状态：`● 运行中 (127.0.0.1:4000)` / `● 已停止`
  - 「重启 codemaker serve」按钮
  - 重启中态：按钮 loading

**直连模式面板**：
- API Base URL 输入框
- API Key 输入框（密码型，可切换显示/隐藏）
- 「测试连接」按钮
  - 测试中：loading
  - 测试成功：`✓ 连接成功，模型已就绪`（绿色提示）
  - 测试失败：`✗ 连接失败：{错误原因}`（红色提示）
- 自定义模型列表（可添加/删除模型名称）

#### 6.3 消耗统计

**正常态**：
- 月份选择器（左右箭头翻页，从 2026-01 起）
- 当前月份：数据卡片展示（Token 消耗 / API 调用次数 / 费用等）
- 数据来源：调用 `codemaker quota` CLI 命令

**加载态**：
- 数据卡片显示骨架屏（Skeleton）

**错误态**（CLI 调用失败）：
- 显示：「获取数据失败，请确认 codemaker CLI 已正常安装」+ 「重试」按钮

#### 6.4 外观设置

**正常态**：
- 主题选择：两个卡片（Light 预览图 / Dark 预览图）
- 当前选中主题有蓝色边框高亮
- 点击即时切换（无需保存按钮，自动持久化）

#### 6.5 数据管理

**正常态**：
- 页面展示危险操作区（红色边框卡片）：
  - 标题：「清空所有数据」
  - 描述：「将永久删除所有会话记录、Agent 配置和本地设置。此操作不可恢复。」
  - 「清空所有数据」按钮（红色 / danger）

**清空确认弹窗**：
- 标题：「⚠ 清空所有数据」
- 描述：「此操作将永久清除以下数据：所有会话记录、消息历史、自定义 Agent 设置、用户设置（主题、Provider 等）。此操作不可撤销。」
- 操作：「取消」+ 「我了解风险，确认清空」（红色 danger 按钮）

## 用户操作路径

### 主路径 1：登录 → 聊天

```
启动应用
  → 检测登录状态
    → 已登录 → 直接进入聊天页（默认激活最近会话）
    → 未登录 → 登录页
                → 点击「使用网易账号登录」
                  → 登录中（loading）
                  → 初始化种子数据（3 个内置 Agent + 示例会话）
                  → 跳转聊天页
```

### 主路径 2：发起一次完整对话

```
聊天页
  → 点击「+ 新建会话」
  → 进入空白对话区（欢迎引导）
  → （可选）点击 Agent 下拉 → 切换 Agent
  → （可选）点击模型下拉 → 切换模型
  → 在输入框输入内容（或拖拽图片 / 粘贴 URL）
  → Enter 发送
  → AI 响应：Thinking Block（折叠）→ ToolCall Block（折叠）→ 最终回复
  → 回复末尾出现快捷选项（若 AI 回复含列表）
  → 继续对话 / 或点击「重新生成」
```

### 主路径 3：安装并使用 MCP

```
左侧导航点击「MCP」
  → MCP 管理页（内置 MCP 已展示）
  → 点击「+ 添加 MCP」
  → 智能安装 Tab：粘贴配置 → 解析 → 确认保存
  → 页面顶部提示「需重启 codemaker serve」→ 点击「立即重启」
  → MCP 状态变为「● 连接中」→「● 已连接 N 个工具」
  → 回到聊天页
  → 输入框 Agent 下拉 → 切换到关联了该 MCP 的 Agent
  → AI 回复中触发 ToolCall Block 展示工具调用过程
```

### 次要路径 1：会话搜索

```
聊天页 → ⌘K
  → 顶部展示搜索输入框（自动聚焦）
  → 输入关键词 → 实时过滤会话列表
  → 点击结果 → 切换到该会话
  → Esc 关闭搜索
```

### 次要路径 2：使用 Skill

```
聊天页输入框 → 输入「/」
  → Skill 选择菜单弹出（向上）
  → 继续输入过滤 / 点击选择
  → Skill 触发词注入输入框提示词
  → 补充具体问题后发送
```

### 异常路径 1：codemaker serve 断线

```
对话中途 → 底部状态栏变红「● 连接失败」
  → 聊天页顶部横幅：「codemaker serve 未运行」
  → 输入区置灰，不可发送
  → 自动尝试重连（状态变为「● 重连中...」）
  → 重连成功 → 横幅消失，恢复正常
  → 重连失败 → 持续显示错误态，提供「前往设置」按钮
```

### 异常路径 2：MCP 需要认证

```
MCP 管理页 → 某 MCP 状态「● 需要认证」
  → 点击「去认证」→ 打开外部浏览器完成认证
  → 认证完成后返回 → 手动刷新或等待轮询更新状态
  → 状态变为「● 已连接」
```
