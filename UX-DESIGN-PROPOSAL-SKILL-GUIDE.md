# UX 设计提案 Skill 使用指南

## 快速开始

### 1. 触发方式

在 Qwen Code 中输入以下任一指令：

```
/ux-design-proposal
```

或使用自然语言：
- "帮我做一个 XXX 的 UX 设计提案"
- "为这个功能生成交互方案"
- "规划 XXX 的页面结构和流程"

### 2. 输入你的想法

描述你的项目/功能想法，越详细越好。例如：

```
我想做一个员工周报系统：
- 每周五下午自动提醒员工填写周报
- 员工可以编辑、提交周报
- 主管可以查看团队周报汇总
- 支持按周/月筛选查看历史周报
- 需要和现有OA系统集成
```

### 3. 选择工作模式

Skill 会询问你希望使用哪种模式：

- **快速模式**：直接生成完整提案（适合快速验证想法）
- **交互模式**：逐段确认，每段生成后你可以调整再继续（推荐）
- **审查模式**：粘贴已有设计稿，输出 UX 评审报告

## 输出内容说明

生成的提案文档包含以下章节：

### 📋 1. 项目概述
- **背景与目标**：为什么要做这个功能/产品
- **核心指标**：如何衡量 UX 成功（如转化率、完成率、满意度）
- **约束条件**：时间、技术、业务限制

### 👥 2. 用户分析
- **用户画像**：2-3 个典型用户角色，包含目标、行为特征、技术熟练度
- **使用场景**：用户在什么情况下会使用这个功能
- **痛点与机会点**：现有方案的问题和新设计的突破点

### 🏗️ 3. 信息架构
- **功能模块图**：功能如何分组和组织
- **页面层级结构**：页面之间的层级关系
- **导航方案**：用户如何在不同页面/模块间跳转

### 🔄 4. 核心交互流程
- **用户旅程地图**：按阶段列出用户行为、系统反馈、情绪变化
- **关键流程图**：使用 Mermaid 语法绘制流程图
- **异常分支处理**：错误、边界、异常场景的交互设计

### 📄 5. 页面规划
- **页面清单**：每个页面的名称、用途、关键组件
- **布局草图**：用文字描述页面区域划分（如：顶部导航 | 左侧菜单 | 主内容区）
- **组件清单**：需要的 UI 组件列表，包含交互说明

### 🎨 6. 设计规范建议
- **色彩方案**：主色、辅助色、状态色建议
- **排版层级**：标题、正文、辅助文字的字号和字重
- **动效建议**：过渡动画、反馈动画、加载状态
- **无障碍设计要点**：对比度、键盘导航、屏幕阅读器支持

### ✅ 7. 验收与度量
- **UX 验收检查清单**：上线前必须通过的 UX 检查项
- **数据埋点建议**：需要追踪的用户行为指标
- **可用性测试方案**：如何验证设计是否好用

## 高级用法

### 审查已有设计

如果你已有设计稿或原型，可以：

1. 粘贴设计稿内容或链接
2. 输入："审查这个设计的 UX"
3. Skill 会输出：
   - 问题清单（按严重程度分级 🔴🟡🟢）
   - 每个问题的具体位置引用
   - 改进建议和参考案例

### 增量补充

对已有提案，可以要求补充特定章节：

- "补充第4章的异常流程"
- "细化页面规划部分，增加移动端适配"
- "补充数据埋点方案"

### 导出和分享

生成的提案是标准 Markdown 格式，可以：

- 直接保存为 `.md` 文件
- 转换为 PDF 分享给团队
- 导入语雀、Notion 等文档平台
- 作为 PRD 附件提交

## 最佳实践

1. **输入越详细，输出越精准**：描述想法时包含目标用户、核心功能、约束条件
2. **使用交互模式**：逐段确认可以及时纠偏，避免生成不符合预期的内容
3. **结合实际技术栈**：告知 Skill 你使用的框架（React/Vue/原生），组件建议会更实用
4. **多次迭代**：第一稿用于验证方向，后续逐步细化特定章节
5. **团队评审**：生成后邀请产品、开发同事一起 review，收集反馈再优化

## 常见问题

**Q: 生成的提案可以直接用吗？**  
A: 可以作为高质量初稿，但建议结合实际业务场景调整。特别是技术可行性、业务规则等需要开发/产品确认。

**Q: 支持移动端设计吗？**  
A: 支持。在输入想法时说明是移动端/响应式/App，提案会包含移动端特定的交互模式（手势、通知、离线等）。

**Q: 能生成 UI 代码吗？**  
A: 当前版本输出的是设计提案文档（文字描述）。如需生成前端代码，可以配合 `frontend-design` skill 使用。

**Q: 如何修改已生成的提案？**  
A: 可以针对特定章节要求重新生成或补充，例如："重新生成第5章，增加权限控制相关的页面"。
